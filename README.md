# 四轨节奏按键点击游戏

## 项目简介

本项目是基于 **MicroPython** 与 **ST7789 显示屏** 的嵌入式四轨节奏按键点击游戏。玩家需要在音符落到判定线的瞬间，按下对应的按键来获得分数。游戏全程包含引导界面、实时计分、判定反馈和结算展示。

## 硬件组成

| 组件 | 型号 / 规格 |
|------|------------|
| 主控芯片 | Raspberry Pi Pico（RP2040） |
| 显示屏 | ST7789 SPI 接口，240×240 像素 |
| 按键 | 4 个独立按键，GPIO 2/1/0/14，上拉输入模式 |
| 通信接口 | SPI0（SCK=GPIO6, MOSI=GPIO8, DC=GPIO7, RST=GPIO11） |

## 功能特性

- **引导画面**：游戏启动后依次展示 5 幅英文说明画面，玩家按任意键翻页，最后自动进入游戏
- **四轨下落**：音符从屏幕上方沿四条轨道随机生成并匀速下落
- **判定线**：屏幕底部设有绿色高亮判定区域
- **计分系统**：分数以 成功击中数-总按键数 格式显示在屏幕顶部
- **游戏结束**：当失误数（总按键数 - 成功击中数）超过 30 时游戏自动结束
- **结算界面**：游戏结束后展示本局的成功击中数与总按键数，按任意键退出程序
- **内存优化**：全程使用局部帧缓冲与手动垃圾回收，适配 72KB 可用 RAM 的极限环境

## 游戏规则

1. 四条轨道从上到下持续随机生成白色音符方块
2. 四个按键从左到右分别对应四条轨道
3. 当音符落到屏幕底部的绿色判定线范围内时，按下对应轨道的按键即判定成功
4. 判定成功：成功击中数 +1，总按键数 +1
5. 判定失败（按下按键但未命中音符）：仅总按键数 +1
6. 当 总按键数 - 成功击中数 > 30 时游戏结束


## 代码结构

| 模块 | 说明 |
|------|------|
| 硬件初始化 | SPI 初始化、ST7789 驱动、按键 GPIO 配置 |
| 帧缓冲 | 音符擦除缓冲、音符绘制缓冲、判定线缓冲、分数缓冲、行文字缓冲 |
| 引导画面 | 5 幅英文操作说明，按任意键翻页，最后自动进入游戏 |
| Note 类 | 音符对象：轨道编号、坐标、存活状态与移动逻辑 |
| 音符生成 | 每 500ms 随机生成 1~2 个音符 |
| 按键检测 | 轮询 4 个按键，检测上升沿触发判定 |
| 判定逻辑 | 检查按键时音符是否在判定线区域内 |
| 分数渲染 | 实时更新并居中显示分数 |
| 结算界面 | 显示本局统计，按任意键退出 |

## 开发环境

- **固件**：MicroPython（RP2040 版本）
- **IDE**：Thonny / VS Code + Pico-Go
- **烧录方式**：通过 USB 将 .py 文件上传至 Pico 文件系统

## 快速开始

1. 将 MicroPython 固件烧录到 Raspberry Pi Pico
2. 安装 ST7789 驱动库（st7789.py）到 Pico 文件系统
3. 按硬件接线图连接显示屏和按键
4. 将 四轨节奏按键点击游戏.py 上传至 Pico 根目录
5. 运行脚本即可开始游戏

> 注意：本程序仅依赖 MicroPython 标准库与 st7789 显示驱动，无需额外第三方库。
